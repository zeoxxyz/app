Engine features

Everything you need to protect your code

Six layers of obfuscation, all configurable from the dashboard. Pick your protection level and ship.

Register-based VM

Your script is compiled into custom bytecode that runs on a polymorphic virtual machine bundled inside the output. The original source never ships.

XOR string encryption

Every string literal in your script is XOR-encrypted at rest and decrypted only inside the VM at runtime — impossible to grep for keys.

Control-flow scrambling

Opaque predicates, dead branches and jump tables shatter the linear flow of your source. Reversers see spaghetti, not logic.

Identifier scrambling

All local variables, parameters and function names are replaced with minified identifiers — while Roblox globals stay untouched and playable.

Anti-tamper checks

Runtime integrity checks and constant-folding traps break any attacker who tries to patch the bytecode or dump memory mid-execution.

Instant builds

Sub-second obfuscation for scripts up to 500 KB. No waiting, no queues — paste, click, ship.

FAQ

Answers, up front

Everything most developers ask before they sign up.

Is ZEOX free to use?

Yes. Every account gets a free daily quota (500 KB obfuscated per day, plus a +100 KB gift). Pro plans unlock higher limits and stronger VM presets.

Does the protected script still run in Roblox?

Absolutely. The output is standard Luau. Roblox globals (game, workspace, Instance.new, etc.) are never renamed, so behavior stays identical.

Can the output be reversed?

Basic minifiers can be undone in seconds. ZEOX ships a full register-based virtual machine — reversing it means writing a devirtualizer for a bytecode that changes with every build.

Do you store my scripts?

Obfuscation runs locally in your browser session. We only store your account info (email + display name) so you can log back in.
